session_cache_limiter()
*использовать с дефолтным агрументом "nocache", однако заголовки посылаемые механизмом сессий могут глушить все прочие,
*подобные конфликты возможно решить, передав в качестве аргумента любой из подоходящих, вплоть до "none", при котором механизм сессий вообще не будет слать заголовки
*ф-цию использовать непосредственно перед session_start()
Убедитесь, что HTTP контент не закэширован для аутентификационной сессии. Допускается кешировать только не конфиденциальный контент. Иначе содержимым могут воспользоваться
http://php.net/manual/ru/session.security.php
...механизм сессий посылает еще и заголовки, запрещающие кэширование страниц (тот самый cache limiter). Для html это правильно и необходимо. Но вот когда вы пытаетесь скриптом, проверяющим авторизацию, отдать файл, то интернет эксплорер отказывается его скачивать. Именно из-за этого заголовка. Вызов session_cache_limiter("private"); перед стартом сессии должен решить проблему
http://phpfaq.ru/sessions#misc
Если страница должна по-разному вести себя для разных пользователей, то кэшировать ее нельзя. Соответственно, старт сессии в дефолтовом варианте с сопутствующей отдачей заголовков с запретом кэширования явно не помешает
http://phpclub.ru/talk/threads/можно-ли-узнать-есть-ли-сессия-не-открывая-ее.9379/page-2
...вам нужно вызывать session_cache_limiter() для каждого запроса (и до вызова session_start()).
http://www.php.su/session_cache_limiter
The only way to remove those headers is to add session_cache_limiter('none'); BUT it only works if you add this line before session_start(). 
http://trac.symfony-project.org/ticket/3542
PHP has horrible misfeature called session.cache_limiter which hopelessly screws up HTTP headers unlesss you set it to none
